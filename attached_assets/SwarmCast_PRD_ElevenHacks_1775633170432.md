## 2. Product Vision

### 2.1 Concept

SwarmCast is a voice-powered public sentiment simulator. A user pastes any document — a press release, policy brief, product announcement, or earnings report — and SwarmCast:

1. **Enriches** it with real web discourse (via Exa) and fact-checks it (via Perplexity).
2. **Generates 25 distinct personas** (via Mistral) with belief states, demographics, and reaction profiles — inspired by MiroShark's agent architecture.
3. **Gives each persona a unique voice** procedurally generated by ElevenLabs Voice Design from their personality profile.
4. **Speaks the swarm**: auto-plays the top persona, and offers a compiled "Focus Group Podcast" audio montage of 5–8 personas reacting in sequence.
5. **Forecasts** 24-hour sentiment trajectory with a confidence band chart.

### 2.2 Why Voice Design (Not Library TTS)

Standard TTS picks from a static list of voices. Voice Design **creates** a voice on the fly from a text description — gender, age, accent, style. This means:

- The skeptical 52-year-old British journalist sounds different from the enthusiastic 28-year-old Brazilian entrepreneur.
- No two analyses produce the same voice set.
- Judges can hear the Voice Design API doing exactly what it was built for.

### 2.3 Goals

- Showcase ElevenLabs Voice Design as the **creative core**, not a report add-on.
- Preserve MiroShark's depth (belief states, web enrichment, persona diversity) in a single-pass, demo-ready architecture.
- Deliver a polished, live-demo-ready experience: paste document → watch pipeline → hear the swarm.

## 3. End-to-End Feature Flow

### 3.1 Phase 1 — Ingest & Context

**User input**

- `title` (string, required)
- `text` (textarea, required) — or text extracted from PDF/TXT file upload
- Optional: file upload (PDF/TXT → text extracted server-side via `pdfminer.six`)

**Step 1: Topic extraction (Mistral)**

Small Mistral call to extract keywords for search context.

- Endpoint: `POST https://api.mistral.ai/v1/chat/completions`
- Model: `mistral-small-latest`
- `response_format: {"type": "json_object"}`
- Output JSON: `{ "keywords": "string", "topic_summary": "string" }`

**Step 2: Exa discourse search (parallel)**

- Endpoint: `POST https://api.exa.ai/search`
- Header: `x-api-key: EXA_API_KEY`
- Request:
```json
{
  "query": "<keywords>",
  "type": "neural",
  "numResults": 8,
  "contents": {
    "text": { "maxCharacters": 500 }
  }
}
```
- Response mapped to:
```ts
type ExaResult = {
  title: string;
  url: string;
  snippet: string;
  publishedDate?: string;
};
```

**Step 3: Perplexity fact-check (parallel with Step 2)**

- Base URL: `https://api.perplexity.ai`
- Endpoint: `POST /chat/completions`
- Model: `sonar-pro`
- Returns `choices[0].message.content` as JSON string, parsed into:
```ts
type FactCheckResult = {
  objective_assessment: string;
  accuracy_rating: "high" | "medium" | "low";
  fact_check_items: {
    claim: string;
    status: "accurate" | "missing_context" | "misleading";
    note: string;
  }[];
};
```

Steps 2 and 3 run in parallel via `asyncio.gather`.

### 3.2 Phase 2 — Persona Swarm (Mistral)

**Persona generation API call**

- Endpoint: `POST https://api.mistral.ai/v1/chat/completions`
- Model: `mistral-medium-latest`
- `response_format: {"type": "json_object"}`
- Context injected: full document text + 8 Exa result snippets + Perplexity fact-check summary

**Persona schema:**
```ts
type Persona = {
  persona_name: string;
  persona_type: "skeptic" | "enthusiast" | "neutral" | "critic" | "concerned" | "influencer" | "institutional";
  background: string;
  mbti: string;
  age: number;
  country: string;
  accent: string;              // e.g., "British", "Brazilian Portuguese", "American"
  gender: "male" | "female";
  voice_style: string;         // e.g., "calm and measured", "rapid and energetic"
  activity_level: number;      // 0.0–1.0
  influence_weight: number;    // 0.5–3.0
  sentiment_bias: number;      // -1.0–1.0
  initial_reaction: string;
  final_opinion: string;
  initial_sentiment: number;   // -1.0–1.0
  final_sentiment: number;     // -1.0–1.0
  belief_state: {
    stance: number;            // -1.0–1.0
    confidence: number;        // 0.0–1.0
  };
  key_concern: string;
  would_share: boolean;
  platform_preference: "twitter" | "reddit" | "both";
  voice_id?: string;           // filled by Voice Design step
  audio_url?: string;          // filled by TTS step
};
```

**System prompt rules (belief state-inspired):**

- Exactly 25 personas; diverse types, ages (18–72), MBTI, regions, accents.
- `final_sentiment` within ±0.3 of `initial_sentiment` unless Perplexity flagged a directly contradicting claim (then up to ±0.5 shift).
- `belief_state.confidence` lower when many fact-check issues found.
- 30% of personas with `would_share = true`.
- At most 6 personas with extreme sentiments (`|final_sentiment| > 0.7`).
- **Each persona must include `accent`, `gender`, and `voice_style` fields** — these are fed directly into Voice Design.

### 3.3 Phase 3 — Voice Design (ElevenLabs) ★ PRIMARY FEATURE

This is the core ElevenLabs integration. For each of the 25 personas, a **unique voice is created** using the Voice Design API.

**Voice Design API call (per persona):**

- Endpoint: `POST https://api.elevenlabs.io/v1/voice-generation/generate-voice`
- Headers: `xi-api-key: ELEVENLABS_API_KEY`, `Content-Type: application/json`
- Request:
```json
{
  "gender": "female",
  "age": "middle_aged",
  "accent": "british",
  "accent_strength": 1.2,
  "text": "This announcement raises serious questions about transparency. The data simply doesn't add up."
}
```

The `text` field is a short sample of the persona's opinion (drawn from `initial_reaction`, trimmed to 50–80 words). The API returns audio bytes for preview + a `voice_id` for use in subsequent TTS calls.

**Accent mapping** (from persona `accent` field to ElevenLabs accent values):

| Persona accent | ElevenLabs accent value |
|---|---|
| American | `american` |
| British | `british` |
| Australian | `australian` |
| African | `african` |
| Brazilian Portuguese | `american` (closest available) |
| Indian | `indian` |
| Irish | `irish` |

**Age mapping** (from persona `age` int to ElevenLabs age category):

| Age range | ElevenLabs age |
|---|---|
| 18–30 | `young` |
| 31–50 | `middle_aged` |
| 51+ | `old` |

**Performance note:** 25 Voice Design calls can be batched in groups of 5 with `asyncio.gather` to respect rate limits while maintaining speed. Store each `voice_id` back to the persona row in the database immediately.

### 3.4 Phase 4 — TTS Audio Generation (ElevenLabs)

After Voice Design creates unique `voice_id`s, use those voice IDs to generate final audio clips.

**Top persona selection for full audio:**

- Compute: `voice_score = |final_sentiment| * influence_weight * (would_share ? 1.3 : 1.0)`
- Sort by `voice_score` descending; select top 8 for full TTS clips.
- Remaining 17 personas have voice IDs (from Voice Design) but no full audio — they can still be listened to on demand via the UI.

**Voice script template:**
> [persona_name], [age], [country]. [initial_reaction] After seeing what people are actually saying — for example, "[Exa result title]" — [my view shifted / I feel even more certain]: [final_opinion] What concerns me most is [key_concern].

Trim to 70–100 words.

**TTS API call:**

- Endpoint: `POST https://api.elevenlabs.io/v1/text-to-speech/{voice_id}`
- Headers: `Accept: audio/mpeg`, `Content-Type: application/json`, `xi-api-key: ELEVENLABS_API_KEY`
- Request:
```json
{
  "text": "...",
  "model_id": "eleven_monolingual_v1",
  "voice_settings": {
    "stability": 0.5,
    "similarity_boost": 0.8,
    "style": 0.0,
    "use_speaker_boost": true
  }
}
```
- Response: binary MP3 audio.
- Save to: `static/audio/{analysis_id}/{persona_id}.mp3`
- Store URL: `/static/audio/{analysis_id}/{persona_id}.mp3`

All 8 TTS calls run in parallel with `asyncio.gather`.

### 3.5 Phase 5 — "Focus Group Podcast" Montage ★ DEMO-DEFINING FEATURE

After individual clips are ready, concatenate them into a single ~90-second audio montage using `pydub`:

```python
from pydub import AudioSegment

def build_montage(audio_paths: list[str]) -> str:
    silence = AudioSegment.silent(duration=400)  # 400ms gap
    combined = AudioSegment.empty()
    for path in audio_paths:
        clip = AudioSegment.from_mp3(path)
        combined += clip + silence
    out_path = f"static/audio/{analysis_id}/montage.mp3"
    combined.export(out_path, format="mp3")
    return f"/static/audio/{analysis_id}/montage.mp3"
```

The montage is the **hero audio feature** — auto-played when analysis completes, and what goes in the demo video. It sounds like a focus group recording.

### 3.7 Phase 7 — Sentiment Forecast

Deterministic local computation (no external API needed — cheaper and faster for demo).

- **Historical points**: pseudo-timeline built from `initial_sentiment` values (20 points across T-20 to T-1 hours).
- **Forecast**: 24 hourly points (T+1 to T+24) using momentum model based on `avg_sentiment`, `viral_potential`, and presence of strong opinions.

```ts
type ForecastPoint = {
  hour_offset: number;       // negative = history, positive = forecast
  sentiment: number;         // -1.0–1.0
  confidence_low: number;
  confidence_high: number;
  is_forecast: boolean;
};
```

## 6. External Service Integrations (Validated)

### 6.1 ElevenLabs — Voice Design (Primary Integration)

- **Endpoint:** `POST https://api.elevenlabs.io/v1/voice-generation/generate-voice`
- **Auth:** `xi-api-key: ELEVENLABS_API_KEY` header
- **Purpose:** Generate a unique voice per persona on the fly from a text description
- **Request fields:** `gender`, `age`, `accent`, `accent_strength` (0.3–2.0), `text` (sample speech, 50–80 words)
- **Response:** Binary audio (preview clip) + `voice_id` in response headers or body
- **Rate limit:** Batch in groups of 5 using `asyncio.gather`

```python
async def create_persona_voice(persona: Persona) -> str:
    age_category = map_age(persona.age)        # "young" | "middle_aged" | "old"
    accent_value = map_accent(persona.accent)   # see accent mapping table
    text_sample = build_sample_text(persona)    # 50–80 word opinion excerpt

    async with httpx.AsyncClient(timeout=30.0) as client:
        resp = await client.post(
            "https://api.elevenlabs.io/v1/voice-generation/generate-voice",
            headers={
                "xi-api-key": os.environ["ELEVENLABS_API_KEY"],
                "Content-Type": "application/json",
            },
            json={
                "gender": persona.gender,
                "age": age_category,
                "accent": accent_value,
                "accent_strength": 1.2,
                "text": text_sample,
            },
        )
    resp.raise_for_status()
    data = resp.json()
    return data["voice_id"]   # Store on persona row
```

## 7. Pipeline Orchestration (`pipeline.py`)

```python
async def run_analysis(analysis_id: UUID, title: str, text: str):
    await db.update_status(analysis_id, "processing")

    # 1) Topic extraction
    keywords = await mistral.extract_keywords(title, text)

    # 2) Exa + Perplexity in parallel
    exa_results, fact_check = await asyncio.gather(
        exa.search_discourse(keywords),
        perplexity.fact_check_document(text),
    )
    await db.save_context(analysis_id, exa_results, fact_check)
    await sse.emit(analysis_id, {"type": "exa_done", "data": exa_results})
    await sse.emit(analysis_id, {"type": "perplexity_done", "data": fact_check})

    # 3) Persona generation
    personas_raw = await mistral.generate_personas(text, exa_results, fact_check)
    personas = await db.insert_personas(analysis_id, personas_raw)
    await sse.emit(analysis_id, {"type": "personas_done", "data": personas})

    # 4) Voice Design — 25 unique voices (batched groups of 5)
    batches = [personas[i:i+5] for i in range(0, 25, 5)]
    for batch in batches:
        voice_ids = await asyncio.gather(*[
            elevenlabs.create_persona_voice(p) for p in batch
        ])
        for p, vid in zip(batch, voice_ids):
            await db.update_persona_voice_id(p.id, vid)
            await sse.emit(analysis_id, {"type": "voice_progress",
                                          "data": {"personaId": p.id, "voiceId": vid}})

    # 5) TTS for top 8 personas
    top_8 = select_top_personas(personas, n=8)
    audio_urls = await asyncio.gather(*[
        elevenlabs.generate_audio(p, str(analysis_id)) for p in top_8
    ])
    await db.update_personas_audio(top_8, audio_urls)

    # 6) Build montage
    montage_url = await montage.build_montage(audio_urls, str(analysis_id))
    await db.update_montage_url(analysis_id, montage_url)
    await sse.emit(analysis_id, {"type": "montage_done", "data": {"montageUrl": montage_url}})

    # 7) Swarm summary
    summary = await mistral.generate_swarm_summary(personas)
    await db.update_analysis_summary(analysis_id, summary)

    # 8) Forecast
    forecast = build_forecast(personas, summary)
    await db.insert_forecast(analysis_id, forecast)

    await db.update_status(analysis_id, "complete")
    full = await db.get_full_analysis(analysis_id)
    await sse.emit(analysis_id, {"type": "complete", "data": full})
```

**Error handling:**

- Fatal errors → set `status = "failed"`, store `error_message`, emit `{type: "error"}`.
- Partial failures:
  - Exa fails → continue without `exa_results` (personas generate from doc only).
  - Perplexity fails → continue without fact-check.
  - Voice Design fails for one persona → retry once; if still fails, assign a fallback library voice.
  - Audio fails for one persona → mark `has_audio = false`, continue.

7. **`pipeline.py`**: Full orchestration with SSE events.
